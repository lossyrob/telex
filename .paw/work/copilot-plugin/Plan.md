# Copilot Plugin Plan

## Approach Summary

Node outcome anchor: complete issue #41 by delivering the Copilot CLI plugin surface that makes the SQLite daemon slice operator-unblocking: installable plugin packaging, single-source skill integration, sessionEnd healthy detach, Copilot-to-generic env mapping, turn-end re-arm guard, removal of generic `COPILOT_*` fallback, and validation evidence. This PR must end with `Closes #41` only if those surfaces are implemented and verified; prerequisite-only or blocker work must use `Refs #41`.

The implementation will keep telex core harness-agnostic by moving all Copilot parsing and env mapping into a hidden Rust adapter command group, `telex copilot ...`, invoked from plugin `hooks.json`. The plugin manifest path is the supported hook path because the extension SDK exposes `onSessionEnd` but not `agentStop`, while Copilot hook configuration supports `agentStop` with `{"decision":"block","reason":...}`.

Planning council findings integrated into this plan:

- Pin turn-guard cap state before coding: store a per-store/session counter in the owner-private daemon run dir, reset it when the session has no unarmed attended stations, and never add daemon IPC state for this node.
- Treat deterministic command behavior as the hard validation gate; treat live Copilot `agentStop` timing/plugin reload proof as a smoke/audit gate if it cannot be fully automated in-session.
- Remove the core `COPILOT_AGENT_SESSION_ID` fallback only together with the Copilot adapter and updated skill guidance, so users have a clear compensating path.
- Update root `SKILL.md` as the single narrative source, and wire the plugin skill to that source without copying divergent content.

Planning review hardening integrated before implementation:

- Resolve the issue wording tension explicitly: this implementation follows `docs/design/daemon.md` and uses daemon `SessionEnd` as the non-destructive healthy-disconnect operation. It releases blocked waiters and marks stations idle; it does not remove membership with literal `Detach`. The PR/Docs.md must call out this semantic resolution so issue #41 is closed on the daemon contract, not by silently changing teardown semantics.
- Empirical `agentStop` timing evidence is a required closure artifact. If a reliable automated harness cannot be built in this session, the PR must include a manual timing probe result or mark the PR partial with `Refs #41`.
- Single-source skill packaging must be chosen and pre-validated before implementation is considered complete. Symlink-dependent packaging is rejected for Windows portability.
- `sessionEnd` store scoping must be explicit. The Copilot adapter resolves the same configured store key as other telex invocations using normal TELEX backend env/config and the daemon cap file for admin proof; if the store/cap/daemon is unavailable, the hook logs a structured no-op and exits successfully without spawning the daemon.
- Turn-guard state and observability are part of the implementation, not deferred polish.

## Key Decisions

- Use Copilot plugin `hooks.json` command hooks rather than `.github/extensions` as the primary integration because issue #41 requires `agentStop` and the SDK extension hooks do not expose that event.
- Implement Copilot mapping in Rust under `telex copilot ...`; do not parse Copilot JSON or read `COPILOT_*` names from generic identity/core helpers.
- Use daemon run-dir state for the turn-guard cap because it is local, owner-private, cross-invocation, and avoids a daemon protocol change.
- `sessionEnd` is best-effort/no-spawn: if the daemon is absent, the hook exits successfully without creating a daemon just to reap nothing.
- The plugin skill must not copy root `SKILL.md`; avoid symlinks and validate the chosen mechanism by content equality against the root file / `telex skill --raw`.
- Plugin hook commands should invoke `telex` by PATH/name rather than pinning an absolute binary path, preserving room for the later seamless-upgrade launcher shim.
- The plugin must not read or write a filesystem session-registry authority; membership remains daemon-native and explicit-only.
